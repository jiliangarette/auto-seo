# Frontend Design Standards

## Before Coding Any UI

Answer these questions:
1. **Purpose** — What is this component/page trying to accomplish?
2. **Tone** — Professional, playful, technical, minimal?
3. **Constraints** — Mobile-first? Accessibility requirements? Brand guidelines?
4. **Differentiation** — What makes this NOT look like every other AI/SaaS template?

## Design Principles

### Typography
- Use distinctive display fonts for headings + refined body fonts
- Establish clear hierarchy with size, weight, and spacing

### Color & Theme
- Cohesive aesthetic with CSS variables
- Dark mode support from the start
- Avoid generic purple/blue AI gradients

### Motion
- Use CSS transitions or React animation libraries
- Prioritize high-impact animations over micro-interactions
- Motion should guide attention, not distract

### Layout
- Spatial composition with intentional whitespace
- Break grid patterns when it serves the design
- Asymmetry can be more engaging than perfect symmetry

## Never
- Generic AI aesthetics (purple gradients, floating particles)
- Inter/Roboto/Arial as the only fonts
- Predictable 3-column card layouts
- Stock icon patterns without customization
- Cookie-cutter templates

## Principle
Match implementation complexity to the vision — maximalist requires elaborate code, minimalist requires restraint.
